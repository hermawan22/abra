# Support

## Questions And Usage Help

Use GitHub Discussions when they are enabled for the repository. If Discussions are not enabled, open a regular issue using the most relevant template and avoid including secrets or private source content.

## Bugs

Open a bug report with:

- Abra version or commit.
- Deployment path: Docker Compose, Kubernetes, Helm, or direct binary.
- Redacted configuration.
- Reproduction steps.
- Redacted logs or command output.

## Security Issues

Do not open public issues for vulnerabilities, leaked secrets, database dumps, embeddings, private source exports, or audit records. Use GitHub private vulnerability reporting when enabled. If it is not enabled, contact the repository owner privately through GitHub before sharing details.

## Scope

The public project can help with the OSS runtime, docs, deployment manifests, and generic extension patterns. Organization-specific connectors, identity integration, source ACL sync, and compliance routing should be handled in deployment overlays.
